Introduktion

I denna modul kan du konfigurera BIND (Berkeley Internet Name Daemon) DNS-servern på denna maskin. DNS-servern skriver om datornamn, t ex www.foo.com, till IP-adresser, t ex 10.1.2.3, och IP-adresser till datornamn.

Grundläggande DNS

Varje DNS-server har hand om ett antal framåt- och bakåtdomäner. En framåtdomän är ett namn, t ex foo.com och en bakåtdomän är ett nummer, t ex 204.165.66. För varje domän kan DNS-servern vara primär server (huvudserver och ansvarig för data om domänen) eller sekundär server (backup för domänen).
Framåtdomäner
Om DNS-servern har hand om en framåtdomän, t ex foo.com, kan den svara på förfrågningar om datornamn i denna domän. För de domäner som denna server är primär server kan du också lägga till datorer och andra uppgifter till domänen i denna modul. För domäner där servern är sekundär server kan du ange primär server (den som servern ska hämta information från).
Bakåtdomäner
Bakåtdomäner kopplar ihop IP-adresser (t ex 203.165.66.77) med datornamn (t ex www.foo.com). En bakåtdomän innehåller datornamn för alla IP-adresser i ett nätverk (t ex 203.165.66). För de domäner som denna server är primär server kan du också lägga till poster med IP-adresser och datornamn i denna modul. För domäner där servern är sekundär server kan du ange primär server (den som servern ska hämta information från).

Sätta upp DNS-servern

Om du redan kör en BIND DNS-server på systemet kan du hoppa över detta. Om Webmin upptäcker att inställningsfilen för den primära DNS-servern (oftast /etc/named.boot) saknas på systemet, antar det att du inte har satt upp någon DNS-server än.

En DNS-server behöver en lista över rotnamnservrar för att svara på DNS-förfrågningar utanför den egna domänen. Om din server inte redan är uppsatt kan Webmin

  1. ladda ned en lista från FTP-servern på rs.internic.net. Om datorn som Webmin körs på är kopplad till Internet - och inte sitter bakom en brandvägg - är detta det bästa alternativet.
  2. använda den lista som skickas med Webmin. Eftersom rotservrar inte ändras så ofta, bör detta fungera.
  3. sätta upp namnserver endast för internt bruk. Om du jobbar på ett nätverk som inte är anslutet till Internet är detta det alternativ du bör välja.

Skapa en primär domän

Fyll i formuläret under Lägg till primär domän på DNS-serversidan för att skapa en ny primär domän. Fälten har följande innebörd:
  • Uppslagning
    Anger om detta är en framåt- eller bakåtdomän.
  • Domän/nät
    För framåtdomäner anges här domän (t ex foo.com). För bakåtdomäner anges här IP-nätet (t ex 203.123.221).
  • Filnamn
    Namn på den fil som domäninformation ska lagras i. Om detta fält lämnas tomt, kommer ett filnamn automatiskt att genereras.
  • E-postadress
    Varje DNS-domän måste ha en e-postadress till den person som ansvarar för domänen.
  • Namnserver
    Detta fält ska innehålla fullständigt namn på den namnserver som hanterar domänen. Det är oftast namnet på den dator du kör Webmin på.
